# Daystart

Bootstrap a new day inside `$LIFEWIKI_VAULT` without depending on Obsidian-rendered query output.

## When to use

Use this skill when the user wants to:
- create today's daily note automatically if it does not exist
- see today's actionable tasks before working
- review unread reading items to choose what to read
- get a Kongfu reminder based on yesterday's note
- write a short startup brief into today's note and receive the same summary in chat

Do not use this skill for ad hoc journaling, inbox capture, or read-it-later ingestion. Use the lower-level Lifewiki skills for those workflows.

## Vault contract

- Vault root: `$LIFEWIKI_VAULT`
- Daily note template: `$LIFEWIKI_VAULT/5-Templates/Daily-Notes.md`
- Daily notes folder: `$LIFEWIKI_VAULT/-Daily-Notes/`
- Daily note filename format: `YYYY-MM-DD.md`
- Read-it-later queue: `$LIFEWIKI_VAULT/Read It Later.md`
- Required authored target heading: `## Notes 📝`
- Query-driven sections that must not be edited directly:
  - `## Today 🔆`
  - `## Dashboard 🗺️`
  - `## Logs 🪵`

## Workflow

1. Resolve today's and yesterday's dates using the local date.
2. Resolve today's daily note path at `$LIFEWIKI_VAULT/-Daily-Notes/YYYY-MM-DD.md`.
3. If today's note is missing:
   - read `$LIFEWIKI_VAULT/5-Templates/Daily-Notes.md`
   - materialize the template for today's date
   - write the result to today's note path
4. Inspect the vault directly for today's tasks:
   - prefer open tasks whose explicit task metadata indicates they happen on or before today
   - include tasks already in progress when that state is visible in raw Markdown
   - if nothing matches, fall back to a short list of open unscheduled tasks
5. Read `Read It Later.md` and collect a short list of unread items.
6. Read yesterday's daily note and inspect `Kongfu::`.
7. If yesterday's note is missing, or `Kongfu::` is missing, empty, or `0`, include a reminder to practice Kongfu today.
8. Write a concise startup summary block under `## Notes 📝` in today's note.
9. Return the same summary in chat and state whether today's note was created during the run or already existed.

## Template materialization rules

- Replace date placeholders with today's actual date values.
- Preserve the template's heading structure and query blocks.
- Do not leave unresolved template placeholders in the created note.
- Do not create a partial raw copy of the template if required substitutions cannot be made clearly.

## Summary format

Write a short authored block such as:

- `### Daystart`
- a `Tasks` subsection with a concise snapshot
- a `Reading` subsection with unread items
- a `Reminder` subsection only when the Kongfu reminder is triggered

Keep the summary brief and useful for immediate action. Preserve wikilinks and task text as written in the vault.

## Failure handling

- If `$LIFEWIKI_VAULT` is unset or unreadable, stop and report it.
- If the daily-note template is missing when today's note must be created, stop and report the expected template path.
- If `Read It Later.md` is missing, stop and report the expected path.
- If yesterday's note is missing, treat that as a reminder condition rather than a hard failure.
- If `## Notes 📝` is missing from today's note, stop and report the required heading.
- If task parsing is uncertain, prefer a smaller trustworthy summary over a speculative or exhaustive one.

## Reporting

After the run, return a short summary that includes:

- whether today's note was created or already existed
- the selected task snapshot
- the selected unread reading snapshot
- whether the Kongfu reminder was triggered
